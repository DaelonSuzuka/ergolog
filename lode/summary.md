# Ergolog — Project Summary

**ergolog** is a minimal, ergonomic Python logging wrapper (v1.0.0, MIT license) by David Kincaid. It wraps Python's `logging` module with a clean API exposed via a single entry point: `from ergolog import eg`. Core features include: named/child loggers via `eg('name')`, a context-manager and decorator tag system (`eg.tag(...)`) with support for positional tags, keyword tags, and callable tag values (e.g. `eg.tag(job=eg.uid)` for auto-generated UUIDs), `eg.counter()` for mutable counter/accumulator tag values that update per-record, an `eg.timer(...)` context-manager/decorator for timing blocks, and an `eg.trace` decorator for function-level tracing. The tag system is thread-safe and async-safe, using `contextvars.ContextVar` for per-context tag isolation — each thread/async-task sees only its own tags. Tags are injected onto `LogRecord` by `ErgoTagFilter` as both `record.tags` (display string) and `record.tag_list` (raw list for structured logging), decoupling them from the formatter. The default config dict is exposed as `from ergolog import config` for easy customization before setup. `dictConfig` only fires if the logger has no existing handlers, so ergolog won't stomp on an application's logging setup. Output is color-coded (ANSI, disable via `ERGOLOG_NO_COLORS` env var) with optional timestamps (disable via `ERGOLOG_NO_TIME`). The default logger name is `ergo` (overridable via `ERGOLOG_DEFAULT_LOGGER`). Built with `uv`, tested with `pytest` + `pytest-cov`, linted with `ruff` (line-length 120, single quotes), and CI runs on GitHub Actions across Python 3.9–3.13.
